# 학습 카드 자가진단 퀴즈

입력: $ARGUMENTS

방금 생성한(또는 지정한) 카드셋을 바탕으로 객관식 8문항 자가진단을 진행합니다.
AskUserQuestion 도구로 한 문항씩 출제하고, 점수 밴드별 피드백을 반환합니다.

---

## 0단계: 입력 해석

$ARGUMENTS 패턴 분기:
- **비어있음** → `$HOME/.claude/cardgen/_temp_card.json` 을 사용 (`/cardgen` 직후 시나리오)
- **파일 경로 (.json)** → 해당 경로의 JSON 사용
- **주제 문자열** → 먼저 `/cardgen` 흐름대로 카드 JSON을 생성해 `_temp_card.json` 에 저장한 뒤 이어서 진행 (HTML 렌더는 생략)

## 1단계: 경로 및 규칙 확인

```
echo "$HOME/.claude/cardgen"
cat "$HOME/.claude/cardgen/rules.md"
```

rules.md의 카드 스키마(concept, key_formula, real_world_usage, example_situation 등)를 숙지하세요. 문항은 이 필드들에서만 파생해야 합니다.

## 2단계: 카드 JSON 읽기

Read 도구로 대상 JSON을 읽으세요.
- 파일이 없거나 `cards` 배열이 비어 있으면:
  > "생성된 카드가 없습니다. 먼저 `/cardgen <주제>` 로 카드를 만들어 주세요."
  메시지 출력 후 종료.

## 3단계: 문항 생성 (엄격한 제약)

다음 규칙을 **정확히** 지킬 것:

- **정확히 8문항** (1~4: 개념 이해, 5~8: 실전 활용)
- 각 문항은 **3~4개의 선택지**, 정답 1개 + **그럴듯한** 오답들
- 문항은 카드의 `concept` / `key_formula` / `real_world_usage` / `example_situation` / `exam_mode` 필드에서만 파생
- 한 문항의 질문은 **1~2문장 이내**
- 선택지는 한 줄씩, 서로 길이 편차가 크지 않게
- **응답 전에 정답을 절대 드러내지 말 것** (해설·정답 표시 금지)
- 카드가 `N`장일 때:
  - `N ≥ 4` → 각 카드에서 1~3문항 고르게 배분
  - `N < 4` → 카드 한 장당 최소 2문항
- `mode` 가 `"시험"` 이면 `exam_mode.likely_question` / `wrong_choices` 활용 가중치 ↑
- `mode` 가 `"면접"` 이면 개념 정의·차이 비교 유형 가중치 ↑

각 문항의 내부 메타:
```
{
  "n": 1,
  "category": "개념 이해" | "실전 활용",
  "question": "...",
  "options": ["A) ...", "B) ...", "C) ...", "D) ..."],
  "correct_index": 0,
  "source_card_index": 0,
  "rationale": "해설(사용자에겐 채점 후 공개)"
}
```

## 4단계: 인터랙티브 출제

AskUserQuestion 도구로 1번부터 8번까지 **순서대로** 제시하세요.

각 호출 포맷:
- question: `"Q{n}/8 [{category}] {question}"`
- multiSelect: false
- options: rules.md의 선택지 그대로, label에 "A", "B", "C", "D" 프리픽스 없이 본문만, description은 비워두기

사용자의 응답을 내부적으로 기록만 하고, **정답 여부와 해설은 공개하지 마세요**. 바로 다음 문항으로 진행.

## 5단계: 채점 및 밴드 판정

8문항 종료 후 점수 집계. 다음 표에 따라 등급과 행동을 정하세요.

| 점수 | 등급 | 다음 행동 |
|------|------|-----------|
| 7~8 | 🌟 마스터 | 다음 주제로 이동 추천 |
| 5~6 | 👍 거의 완료 | 틀린 문항의 출처 카드 복습 |
| 3~4 | 📚 성장중 | 카드셋 전체 재학습 후 재시도 |
| 0~2 | 🔁 재시작 | 난이도를 낮추거나 선수 개념 학습 |

## 6단계: 결과 및 오답 피드백 출력

다음 형식으로 한 번에 출력:

```
=== 자가진단 결과 ===
점수: N / 8  ·  등급: {밴드}
개념 이해: a / 4  ·  실전 활용: b / 4

틀린 문항:
- Q{n} ({category})
  - 사용자 선택: {선택지 본문}
  - 정답: {정답 본문}
  - 해설: {rationale}
  - 출처: 카드 {source_card_index+1} — "{cards[source_card_index].question}"
    핵심 원리: {concept.core_principle 한 줄 요약}

집중 복습이 필요한 카드:
- 카드 {i}: {question}  (틀린 문항 수 {k})
```

틀린 문항이 0개면 "틀린 문항 없음" 으로만 표기.

## 7단계: 다음 단계 제안

다음 세 가지를 옵션으로 제시:
1. **재시도** — 같은 카드셋으로 새 문항 8개 생성
2. **다른 주제 퀴즈** — `/cardgen-quiz <주제>` 안내
3. **특정 카드 심화 설명** — 사용자가 카드 번호를 말하면 `concept` 5개 필드 전부 풀어서 설명

AskUserQuestion으로 선택받아도 되고, 단순 안내만 해도 됨.

---

## 오류 처리

- JSON 파싱 실패 → rules.md의 스키마로 재검증, 누락 필드는 빈 문자열 처리
- AskUserQuestion 도구 미가용 환경 → 텍스트 기반 폴백: 한 문항씩 출력 후 "A/B/C/D 중 선택" 프롬프트
- 카드 수가 1장이면 → 문항 수를 4개로 축소하고 그 사실을 상단에 안내
